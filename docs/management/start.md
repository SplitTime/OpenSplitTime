# Starting

## Starting: Overview

To start an Event, you will follow these steps:

1. Check in Entrants

1. Make modifications as needed

1. Verify bib numbers

1. Start Entrants

## Checked In Status

Each Entrant is either "checked in" or "not checked in." Entrants who are checked in can be started using the
[Start Entrants](https://www.opensplittime.org) process.

**Entrants that are not checked in will not be started using the normal Start Entrants process.**

## Scheduled Start Times

In addition, each Entrant has a "scheduled start time." If the scheduled start time for an entrant is not
specified, OST will use the Event start time by default. You will start your entrants in batches depending on
scheduled start time.

You will have the opportunity to enter an actual start time for each batch of Entrants at the time you start your
Entrants. This is useful if your Event starts a few minutes late, as is often the case.

## Set Up the Check-In Table

Starting an Event Group begins at check-in time. We recommend you place a laptop or tablet at the check-in table
where bibs are handed out and then follow this process:

1. Open a browser

1. Log in to OpenSplitTime.org using an account that has Owner or Steward privileges for your Organization

1. Visit the Roster page (**Admin > Roster**)

## Check In Entrants

As each runner arrives, find the runner's name in the Roster and click/tap "Check In." The button will
change from white to green indicating the check-in was successful. **Make sure the runner is checked in to
OST before handing the bib to the runner.** If a runner receives a bib without being checked in to OST,
that runner will not be started when
you [start efforts](https://www.opensplittime.org).

## Changing Entrant Information

Sometimes you will need to make changes to an Entrant's information at the last minute. You can do this during
the check-in process (or at any other time) from the Construction Entrants view.

1. Go to the Construction Entrants view (**Admin > Construction > Entrants**)

1. Find the entrant in the list

1. Click the **Actions** kebab menu and click **Edit**

1. Make the needed changes

1. Click **Update Entrant**

OpenSplitTime will update the Entrant. If an update is not possible (for example, because the bib number was
changed to a bib number already assigned to another Entrant), then OpenSplitTime will not update the Entrant
and will alert you of the problem.

## Moving an Entrant to Another Event

If your Event Group includes more than one Event, you may need to change an Entrant from one Event to another.
You can do this from the Construction Entrants view.

1. Go to the Construction Entrants view (**Admin > Construction > Entrants**)

1. Find the entrant in the list

1. Click the **Actions** kebab menu and click **Edit**

1. Use the **Event** dropdown menu to select the correct Event

1. Click **Update Entrant**

OpenSplitTime will update the Entrant. If an Entrant has times already recorded, OpenSplitTime will attempt to
change those times to the newly assigned Event. If times are incompatible (for example, if the Entrant has a time
entered at Aid 3 but the Event to which the Entrant is being moved does not go to Aid 3), then OpenSplitTime will
not update the Entrant and will alert you of the problem.

## Deleting an Entrant

You can also delete an Entrant from the Construction Entrants view.

1. Go to the Construction Entrants view (**Admin > Construction > Entrants**)

1. Find the entrant in the list

1. Click the **Actions** kebab menu and click **Delete**

1. Confirm your choice by clicking **OK**

OpenSplitTime will delete the Entrant.

## Starting Entrants

Entrants will be shown as "Not Started" until they are started&mdash;in other words, until they have a recorded
Split Time. Starting an Entrant records a Split Time at the starting split. Entrants are started using one of the
tools provided in the Roster view.

Note: **If an Entrant is not started and has no other recorded Split Times**, OpenSplitTime will
report that Entrant to OST Remote as "Not Expected," and
**the Entrant's bib number will appear black (Not Expected) in the OST Remote Cross Check view**.
Make sure to timely start your Entrants so that stations using OST Remote will have an accurate Cross Check.

## Starting Entrants In Bulk

The Roster view includes a green button that will initially indicate that there is
**Nothing to Start**. Remember that each Entrant has a "scheduled start time", as discussed on the
[Starting Overview](https://www.opensplittime.org) page. An Entrant becomes eligible
to start when the following are true:

1. The Entrant has checked in

1. The Entrant's scheduled start time is earlier than the current time

If any Entrant meets both criteria, the text of the green button will change to **Start Entrants**.
To start Entrants, do the following:

1. Click **Start Entrants**

1. Choose the applicable scheduled start time from the dropdown menu. A dialog box will appear.

1. If the actual start time is different from the scheduled start time, change the actual start time in the
dialog box.

1. Click **Start**

## Starting a Single Entrant

Sometimes you may need to start a single Entrant. You can do this from the Show Entrant page.

1. Click on the Entrant's name

1. Click **Start Effort**

## Changing Start Times

Sometimes you may start entrants with the wrong start time. For example, you may have started entrants with a
start time at the exact scheduled start time of 07:00:00, and later you may determine that the actual start time
was a few seconds later, at 07:00:04.
**You can change start times in bulk using the Manage Start Times view.**

## Manage Start Times View

You can reach the Manage Start Times View by going to **Admin > Construction > Entrants > Actions > Manage
Start Times**.

In the view, you will see all efforts listed for your event group. The efforts will be grouped by event and by
actual start time. This view allows you to change, in bulk, start times for all efforts having a given actual
start time.

For example, assume you want to change all efforts in a particular event having an actual start time
of 07:00:00 to 07:00:04. Do this as follows:

1. Go to the Manage Start Times view

1. Find the group of efforts for the event in question having an actual start time of 07:00:00 (the date will be
shown as well)

1. In the header for that group, click the blue pencil icon

1. Enter the corrected actual start time (including the date)

1. Click the submit button (the button with the check mark)

The actual start time for all efforts in that group will be updated to reflect the corrected time.
